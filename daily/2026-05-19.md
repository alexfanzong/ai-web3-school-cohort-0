# Daily Note: 2026-05-19

## Topic

From chatbot to Agent.

## Summary

Today's useful frame was the Agent loop: goal understanding -> tool use -> observation -> repair. The practical value is not "AI chats better", but that AI can operate workflows across local tools, memory, skills, and external channels.

## My Takeaway

For Web3 compliance work, the important design question is not whether an Agent can act. It is which actions it may take without human review, and which actions must stop at a confirmation boundary.

## Programmable Compliance Connection

Agent workflows should expose permission boundaries before execution:

- low-risk: summarize, classify, draft, retrieve, compare;
- medium-risk: prepare transaction intent, generate review checklist, request evidence;
- high-risk: sign, pay, settle, approve legal conclusion, or bind a user commitment.
