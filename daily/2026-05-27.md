# Daily Note: 2026-05-27

## Topic

Neo-Cypherpunk and the cultural layers of privacy.

## Summary

今天听了 Peter Farbey 关于 Neo-Cypherpunk 与 Web3 隐私文化的分享。整体感受是，这场更像一场价值观和文化层面的导入，而不是具体技术方案或产品方法论。它强调隐私不是少数黑客、密码学家或极客圈子的专属议题，而应该成为普通用户、builder、社区和协议共同参与的公共能力。

比较有启发的一点是，讲者没有把隐私讲成“完全匿名”或“彻底脱离大平台”，而是强调 privacy stack 可以是渐进式的。不同人有不同风险模型，不需要一上来就追求 100% 隐私。对普通用户来说，从换浏览器、使用加密通信工具、理解钱包和链上痕迹开始，本身就是一种隐私意识训练。

## Key Ideas

- Neo-Cypherpunk 试图把早期 cypherpunk 传统更新成更开放、更低门槛、更社区化的隐私文化。
- 隐私不是一个单点工具，而是一组持续选择，覆盖浏览器、钱包、通信工具、文档、设备和链上行为。
- 不存在一套适用于所有人的 universal privacy stack。记者、公众人物、普通用户、AI agent 产品面对的是不同风险模型。
- AI agent 会带来新的隐私暴露面，因为它可能替用户交易、通信、调用工具，并留下可观察的行为轨迹。

## My Takeaway

我也觉得这场分享的实质内容偏理想化。很多部分停留在文化动员、工具推荐和社区语言，没有展开 AI agent、链上交易隐私、合规冲突这些更硬的问题。

但它仍然提出了一个值得 AI x Web3 builder 继续追问的问题：当 agent 开始替用户行动时，什么东西会变得可见？交易对象、交易频率、调用工具、支付流、跨平台行为，都可能变成可画像的信号。隐私不能再被当成产品完成后才补的偏好设置，而应该从钱包、agent runtime、协议和审计设计阶段就进入架构。

## Programmable Compliance Connection

对我的 Programmable Compliance 方向来说，这场分享的价值在于提醒我：隐私和合规不应该被简单看成对立关系。

差的合规设计会因为追求安全感而收集过多原始数据。更好的合规设计应该在最小披露的前提下完成必要验证。放到 AI x Web3 里，需要继续追问：

- Agent 应该证明什么，而不暴露完整底层数据？
- 哪些用户属性可以 selective disclosure？
- 哪些交易需要可审计，审计对象是谁？
- 什么时候隐私应该默认开启，什么时候透明性应该由用户主动选择？

这指向一个更精确的可编程合规版本：不是“先收集一切，事后再检查”，而是在执行前定义最小必要证明、披露边界和审计路径。
