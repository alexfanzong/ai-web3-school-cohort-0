# Daily Note: 2026-05-24

## Topic

Ethereum x AI Agent Economy.

## Summary

Today's session framed AI agents as emerging economic participants rather than only productivity tools. If agents can write code, make decisions, hire other agents, and generate income, they need infrastructure for identity, reputation, payment, constraints, and settlement.

The key thesis: Ethereum can become a coordination and settlement layer for an open AI agent economy.

## Key Primitives

- ERC-8004: agent identity and reputation.
- ERC-8003 / x402: machine-to-machine payment through HTTP 402 and stablecoin settlement.
- Smart contract wallets: programmable spending limits, merchant restrictions, and execution constraints.
- Deep Funding: using agents to scale human judgment in public goods allocation, without letting agents replace human value-setting.

## My Takeaway

The most important distinction is that programmable money is not just crypto payment. The money itself can carry constraints. For AI agents, this is the difference between giving an agent a credit card and giving it a bounded wallet with explicit rules.

That maps directly to my Programmable Compliance thesis: constraints should be embedded into the transaction substrate before risky actions happen.

## Programmable Compliance Connection

Agent economy makes compliance design more concrete:

- Agent identity and reputation matter before interaction.
- Payment constraints should be embedded before execution.
- Human judgment should be scaled, not replaced.

For my own direction, the key question is how to make agent authority legible without pretending that every compliance judgment can be fully automated.
