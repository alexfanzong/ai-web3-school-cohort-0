# Personal Learning Plan

Version: 2026-05-18 learner-tailored draft.

## Goal

Build a public proof-of-work trail for AI x Web3 School: understand the shared concepts, complete daily learning notes, turn questions into Handbook feedback, and shape one legal/compliance-architecture-led hackathon prototype.

## Core Thesis

Programmable Compliance / 可编程合规: compliance should be an upfront product and protocol design layer, not merely an after-the-fact legal remedy.

For this learning track, every AI x Web3 topic should be tested against one question: how can compliance constraints become visible, programmable, reviewable, and enforced before risky user, agent, wallet, or protocol actions happen?

## Current Assumptions

- AI workflow: can use AI agents for vibe coding and project building.
- Background: social science, no traditional programming foundation.
- Web3 base: product architecture experience in cross-border legal/compliance contexts.
- Direction: Web3 cross-border legal/compliance architecture, RWA, DAO, Agentic Commerce, product research, and hackathon participation.
- Time budget: 1-2 hours per day.
- Output language: Chinese first, with English terms and evidence when useful.

This plan should optimize for concept clarity, compliance architecture, product judgment, workflow design, and demo-quality proof-of-work. Coding should be agent-assisted and scoped to small prototypes instead of becoming the main bottleneck.

## Product Research Outputs

For each major topic, prefer one of these outputs:

- Problem memo: who has the problem, when it appears, why current tools fail.
- Workflow map: user intent -> agent action -> wallet/tool permission -> human confirmation -> record/audit.
- Risk note: what could go wrong, what must never be automated, what needs rollback or review.
- Prototype brief: target user, core loop, demo script, data needed, success criteria.
- Compliance architecture note: parties, assets, fund flow, jurisdiction assumptions, regulated touchpoints, prohibited automation, human review points.
- Programmable compliance reflection: what should be visible, reviewable, and constrained before execution.

## Hackathon Bias

Good first directions for this profile:

- Agentic Commerce Compliance Layer: agent-assisted commerce workflow with payment/settlement, jurisdiction checks, user disclosures, and human approval before binding actions.
- RWA Compliance Copilot: classify asset, jurisdiction, participant, transfer restriction, and required disclosure/evidence in a transparent workflow.
- DAO Governance Legal Intelligence: summarize proposals, compare stakeholder arguments, identify governance/legal risk, and create transparent voting-assist briefs.
- Programmable Compliance thesis: explore how compliance can become an upfront design constraint without publicly exposing detailed mechanics too early.
